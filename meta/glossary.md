# Glossary

Short, plain-language definitions of terms used across the course. Keep entries one or two
sentences; link to the chapter that teaches each term in depth. Alphabetical.

- **ACID** — Atomicity, Consistency, Isolation, Durability; the correctness guarantees of
  transactional databases.
- **Cache hit / miss** — whether requested data was already in the cache (hit) or had to be
  fetched from the slower source (miss).
- **CAP theorem** — under a network partition a distributed system must choose between
  consistency and availability.
- **Latency** — time to handle a single request, end to end.
- **Throughput** — number of requests handled per unit time.
